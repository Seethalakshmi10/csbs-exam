FIREBASE SETUP GUIDE
Connect students across all devices · Free · 10 mins
HOW IT WORKS
👨‍💻
Student opens HTML on their device

☁️

🖥️

STEP 1 — CREATE FIREBASE PROJECT (FREE)
1
Go to Firebase Console
Open console.firebase.google.com and sign in with your Google account.
2
Create New Project
Click "Add project" → Name it csbs-exam → Disable Google Analytics → Click "Create project".
3
Add a Web App
In your project, click the </> (Web) icon → Register app as csbs-exam-web → Copy the firebaseConfig object shown.
4
Enable Firestore Database
In left sidebar → Build → Firestore Database → Click "Create database" → Choose "Start in test mode" → Select your region → Done.
STEP 2 — PASTE CONFIG INTO THIS FILE
Open this HTML file in a text editor (Notepad, VS Code). Find this section near the top and replace with your actual values:
const firebaseConfig = { apiKey: "YOUR_API_KEY", // ← paste your value authDomain: "YOUR_PROJECT.firebaseapp.com", projectId: "YOUR_PROJECT_ID", // ← e.g. "csbs-exam-abc12" storageBucket: "YOUR_PROJECT.appspot.com", messagingSenderId: "YOUR_SENDER_ID", appId: "YOUR_APP_ID" };
💡 Your config looks like: apiKey: "AIzaSy...", projectId: "csbs-exam-abc12", etc.
STEP 3 — HOST THE FILE (SHARE WITH STUDENTS)
Students need to open the same HTML file. Pick the easiest hosting method:
EASIEST · FREE
📄 Google Drive
Upload HTML to Drive → Right click → Share → "Anyone with link" → Copy link → Send to students
⚠️ Drive preview won't work — students must click "Open with → Chrome"
BEST · FREE
🌐 GitHub Pages
Create free GitHub account → New repo → Upload HTML as index.html → Enable Pages → Share the URL
✓ Proper URL, works on any device, mobile-friendly
INSTANT · FREE
⚡ Netlify Drop
Go to app.netlify.com/drop → Drag and drop the HTML file → Get instant URL in seconds
✓ Fastest option, no account needed
WIFI ONLY
📡 Local Network
Run a local server on your laptop and share IP. All students must be on the same WiFi network.
Use: python3 -m http.server 8080
⚠️ FIRESTORE SECURITY RULES (IMPORTANT)
In Firebase Console → Firestore → Rules tab, paste these rules to allow students to write but not read each other's data:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /participants/{pid} { allow write: if true; // students can submit allow read: if false; // students cannot read others } } }
Admin reads happen through the Firebase console / admin SDK. For this app, admin login reads all data — you can tighten rules later.
